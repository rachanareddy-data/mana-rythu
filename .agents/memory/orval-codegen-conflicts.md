name: Orval codegen conflict fix
description: How to prevent duplicate export conflicts between api.ts and types/ generated by Orval
---

## Rule
Never use inline `type: object` schemas directly in OpenAPI path `requestBody`. Always reference a named component schema via `$ref: "#/components/schemas/NamedSchema"`.

**Why:** Orval generates TypeScript types for inline schemas (e.g., `AdminVerifyUserBody`) in BOTH `api.ts` AND `types/`. This causes a duplicate export error in `lib/api-zod/src/index.ts` which re-exports both files.

**How to apply:** Any time you add a `requestBody` to an OpenAPI operation, define the input schema in `components/schemas` with a unique name, then reference it via `$ref`. Never write `schema: { type: object, properties: {...} }` inline.

**Note:** `lib/api-zod/src/index.ts` is fully regenerated by Orval — do not manually edit it to fix conflicts. Fix the spec instead.
